# Agile Workflows & Kanban

## 1. Research & Learning

### How a Kanban Board Works

A Kanban board is a visual workflow management tool used in Agile teams to track tasks and manage work progress. It provides a clear overview of all tasks in a project and shows the current status of each task.

Each task is represented as a card on the board, and the board is divided into columns that represent different stages of the workflow. As work progresses, tasks move from one column to another until they are completed.

Kanban helps teams:

- visualize work clearly
- track progress in real time
- identify bottlenecks
- improve workflow efficiency

By providing transparency and structure, Kanban helps teams manage their workload and deliver tasks more effectively.

---

### Kanban Columns and Their Meaning

A Kanban board typically contains several columns representing different stages of the development process. In my internship repository Kanban board, I structured the workflow using the following columns:

**Not Started**

This column contains tasks that have been identified but have not yet begun. These tasks are planned and prioritized before work starts.

**In Progress**

Tasks move to this column when development or implementation work has started. This indicates that a team member is actively working on the task.

**Ready for Review**

When a task is completed by the developer, it moves to this column for verification. This stage ensures that the work meets the required standards before final approval.

**Approved by Bot**

Once the work has been reviewed and confirmed to meet the project requirements, the task is marked as completed and moved to this column.

These columns help clearly represent the **status and progress of each task** throughout the workflow.

---

### How Tasks Move Through the Kanban Board

Tasks move through the Kanban board as work progresses. The general workflow process is:

1. A task is created and placed in **Not Started**.
2. When work begins, the task is moved to **In Progress**.
3. After the work is completed, the task moves to **Ready for Review**.
4. Once the task is verified and approved, it is moved to **Approved by Bot**.

Responsibility for updating task status usually belongs to the team member currently working on the task. Maintaining accurate status updates ensures the Kanban board reflects the real-time progress of the project.

---

### Benefits of Limiting Work in Progress (WIP)

Limiting Work in Progress (WIP) is an important principle of Kanban. It restricts the number of tasks that can be actively worked on at the same time.

Key benefits include:

**Improved focus**

Team members can concentrate on completing current tasks rather than starting too many tasks simultaneously.

**Reduced task switching**

Working on fewer tasks reduces context switching, which increases productivity and work quality.

**Faster completion**

By finishing tasks before starting new ones, work moves through the workflow more efficiently.

**Better workflow visibility**

WIP limits help identify bottlenecks in the process and allow teams to adjust workload accordingly.

# 2. Reflection

### How Kanban Helps Manage Priorities and Avoid Overload

Kanban helps manage priorities by clearly displaying all tasks and their current status on a single board. This visual representation allows team members to easily understand which tasks are most important and what work is currently being done.

By limiting the number of tasks that can be in the **In Progress** column, Kanban prevents team members from taking on too much work at the same time. This helps avoid overload and ensures tasks are completed before new work is started.

Additionally, Kanban supports continuous workflow management, allowing tasks to be prioritized and adjusted dynamically based on project needs.

---

### How I Can Improve My Workflow Using Kanban Principles

Using Kanban principles helps me organize tasks more effectively and maintain clear visibility of my work progress during the internship.

Firstly, I can prioritize tasks more carefully before moving them into the **In Progress** stage. This ensures that I focus on high-value tasks that contribute directly to project goals.

Secondly, I can update the board consistently so that the status of each task accurately reflects my progress. This helps maintain transparency for team members and supervisors.

Finally, applying Work in Progress limits will help me focus on completing tasks sequentially rather than starting multiple tasks at once.

These improvements will help increase my productivity and ensure a more structured workflow.
