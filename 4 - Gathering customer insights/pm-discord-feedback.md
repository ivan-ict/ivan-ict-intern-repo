# Analysing Customer Feedback in Discord

**Milestone:** Gathering Customer Insights
**Task:** Analysing Customer Feedback in Discord (#17)

---

## Goal

The goal of this task is to review user feedback and discussions in Discord and onboarding feedback records to identify common feature requests, pain points, and positive feedback. This helps the team understand user needs and improve the product based on real user experiences.

---

## Why is this important?

Community platforms like Discord provide raw and honest feedback from users. Unlike formal surveys, users freely share their frustrations, suggestions, and positive experiences. Product Managers can analyse this feedback to identify patterns, prioritize features, and improve user experience.

---

## Research & Learn

### Why do users prefer community platforms like Discord for feedback?

Users prefer Discord because:

- It allows real-time communication with developers and other users.
- Users feel more comfortable sharing honest feedback in a community.
- They can see solutions and discussions from other users.
- It creates a sense of community and belonging.
- Users can suggest features and discuss ideas interactively.

### How do companies track informal user feedback?

Companies usually track informal feedback by:

- Monitoring Discord channels and tagging feedback messages.
- Recording feedback into spreadsheets or project management tools.
- Creating GitHub issues based on user feedback.
- Categorizing feedback into bugs, feature requests, UX issues, etc.
- Looking for repeated feedback to identify common problems.

### What are common challenges in interpreting user feedback?

Some challenges include:

- Users describe problems differently, making it hard to categorize.
- Some feedback is emotional rather than specific.
- Not all feedback represents the majority of users.
- Some users request features that do not align with the product roadmap.
- It can be difficult to distinguish between user error and actual bugs.

---

## Recurring Requests / Pain Points Identified

After reviewing customer feedback and onboarding notes from the past month, several recurring issues and feature requests were identified:

### 1. Onboarding Process is Confusing or Too Long

Many users mentioned that onboarding was too long, confusing, or overwhelming. Some users wanted a simpler onboarding process with fewer steps and clearer instructions.

**Examples:**

- Users felt the setup process took too long.
- Some users skipped onboarding because it was overwhelming.
- Users suggested a more guided onboarding process.

### 2. Website/App Blocking Not Working Properly

Several users reported that website blocking did not work correctly during focus sessions or onboarding.

**Examples:**

- Blocking did not start automatically.
- Some websites were not blocked even when added to the block list.
- Blocking sometimes required manual focus session start.

### 3. UI/UX Issues and Accessibility Problems

Users reported issues related to UI design, font size, pop-ups, and layout problems.

**Examples:**

- Font size too small.
- Pop-ups appearing behind windows.
- UI looked too technical or not user-friendly.
- Some buttons were hard to find.

### 4. iOS Installation and TestFlight Issues

Many users had problems installing the iOS version using TestFlight.

**Examples:**

- Redeem code issues.
- TestFlight not accepting new testers.
- Confusion about how to install the iOS app.

---

## Interesting User Suggestion Analysis

### Suggestion: Automatic Website Blocking Based on User Intent

#### What problem does it solve?

This solves the problem where users forget to manually add distracting websites to the block list. Automatic detection would make the app smarter and reduce setup time.

#### Would this fit Focus Bear’s roadmap?

Yes, this fits Focus Bear’s goal of using AI to help users stay focused. Intelligent automatic blocking aligns with the product’s AI-based focus assistant concept.

#### What challenges might arise from implementing it?

- AI might incorrectly block useful websites.
- Requires accurate AI detection and classification.
- Privacy concerns if the app monitors browsing activity.
- Development complexity and testing required.

## Reflection

### What were three common user concerns or feature requests?

The three most common concerns were:

1. Onboarding process is too long or confusing.
2. Website/app blocking does not always work properly.
3. UI/UX improvements such as font size, layout, and pop-ups.

### How does user sentiment in Discord compare to support tickets?

Discord feedback is usually more informal and honest. Users openly share frustrations and suggestions. Support tickets are more structured and focused on specific technical issues. Discord is better for identifying patterns and feature ideas, while support tickets are better for identifying specific bugs.

### What’s one user suggestion that stood out to you, and why?

The suggestion about automatic website blocking stood out because it aligns strongly with the main purpose of Focus Bear, which is helping users stay focused. If implemented well, this feature could significantly improve user experience and make the app smarter and more effective.

---

## Summary

From analysing customer feedback, the main areas for improvement are onboarding experience, blocking functionality, UI/UX design, and iOS installation process. User feedback is very valuable because it shows real user needs and helps the team prioritize future improvements.
