# Reviewing Support Tickets in Zoho Desk (#18)

## Overview

This task focuses on analysing customer support tickets from Zoho Desk to identify common issues, user pain points, and actionable product insights.

---

## Goal

Learn how to:

- Review support tickets effectively
- Categorise customer issues
- Extract meaningful insights for product improvement

---

## Why This Is Important

Support tickets provide **direct user feedback**, often highlighting:

- Bugs affecting user experience
- Missing or desired features
- Usability problems

For Product Managers, this data is critical for:

- Prioritising the backlog
- Improving user experience
- Identifying recurring problems

---

## Research & Key Learnings

### 1. Common Types of Customer Issues

From reviewing tickets, issues generally fall into:

- **Bugs**
    - App crashes
    - Sync issues
    - Login inconsistencies
    - Subscription/payment errors

- **Feature Requests**
    - New functionality (e.g., better analytics, reminders)
    - Integration requests
    - Customisation options

- **Usability Issues**
    - Confusing UI/UX
    - Poor onboarding
    - Lack of clear instructions

---

### 2. How PMs Use Support Tickets

Product Managers use support tickets to:

- Detect **recurring issues** (high-impact problems)
- Identify **user needs and expectations**
- Validate **product decisions with real data**
- Feed insights into frameworks like **RICE prioritisation**

---

### 3. Categorisation System

A simple and effective system:

| Category        | Description                            |
| --------------- | -------------------------------------- |
| Bug             | Something not working as intended      |
| Feature Request | User asking for new functionality      |
| Usability Issue | Confusing or difficult user experience |

### Recurring Themes

#### 1. Subscription & Sync Issues

- Users report:
    - Subscription active but not reflected in app
    - Restore purchase not working
- Impact:
    - High frustration
    - Directly affects revenue & trust

#### 2. Confusing User Experience (UX)

- Users struggle with:
    - Navigation between screens
    - Understanding features (e.g., stats, settings)
- Impact:
    - Drop-off during onboarding
    - Reduced engagement

## Example Ticket Analysis

### Problem

User reports:

> Subscription is active, but the app shows no access. Restore purchase fails and resubscribe shows already subscribed.

### Support Handling

- Asked user to:
    - Try restoring purchase
    - Restart app
- Issue remained unresolved

### Root Cause (Likely)

- Backend sync issue between subscription service and app state
- Poor error handling/messages

### Prevention Suggestions

- Improve subscription state validation
- Add clearer error messages
- Implement auto-sync on app launch
- Provide in-app troubleshooting guidance

---

## Reflection

### 1. Common Pain Points

- Subscription/payment inconsistencies
- Confusing UI and unclear flows

---

### 2. Patterns in User Frustration

- Users feel frustrated when:
    - Features don’t work as expected
    - They don’t understand what to do next
- Repeated complaints indicate **systemic issues**, not isolated cases

---

### 3. Recommendations for Focus Bear

#### Product Improvements

- Fix subscription syncing reliability (high priority)
- Improve onboarding clarity
- Simplify navigation between key screens

#### Process Improvements

- Tag and track recurring issues in Zoho Desk
- Integrate support insights into backlog prioritisation
- Regularly review support data (weekly)

---

## Key Takeaways

- Support tickets are a **goldmine of product insights**
- Recurring issues = **high priority fixes**
- Clear categorisation helps identify patterns quickly
- PMs should continuously connect **user feedback → product decisions**
