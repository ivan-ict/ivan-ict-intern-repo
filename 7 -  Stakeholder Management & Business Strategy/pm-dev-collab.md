# Working with Developers (#6)

## Overview
This task focuses on understanding how Product Managers collaborate with developers to ensure features are clearly defined, feasible, and efficiently implemented.

---

## Goal
To improve communication between PMs and developers by understanding their challenges, expectations, and workflow.

---

## Why This Is Important
Effective collaboration ensures:
- Features are built correctly the first time
- Miscommunication is reduced
- Development time and effort are optimised

PMs act as a bridge between **business goals and technical execution**.

---

## Research & Learn

### 1. Challenges Developers Face with PMs
- Unclear or ambiguous requirements
- Frequent changes in scope
- Lack of technical consideration
- Missing edge cases or acceptance criteria

---

### 2. Best Ways to Clarify Requirements
- Define clear problem statements
- Provide concrete examples and use cases
- Include acceptance criteria
- Use visuals (wireframes, flows)

---

### 3. Handling Constraints & Trade-offs
PMs and developers collaborate to:
- Balance **impact vs effort**
- Identify **technical limitations early**
- Adjust scope to deliver value faster

---

## Developer Feedback (Focus Bear)

### What makes working with PMs easier?
> “Clear requirements and examples help a lot. When we understand exactly what problem we are solving, it’s much easier to implement.”

---

### What frustrates developers about PMs?
> “Sometimes requirements are too vague or change during development, which causes rework and confusion.”

---

## Feature Spec Improvement

### Improvement Identified
Add **clear acceptance criteria and edge cases** to feature specs.

### Why This Helps
- Reduces ambiguity
- Minimises back-and-forth communication
- Helps developers test their implementation
- Improves alignment between PM, QA, and engineering

### Example Improvement

Instead of:
> “User should be able to access stats”

Better:
- User can access stats only when authenticated
- If not authenticated → show login prompt
- If session expired → show error + retry option

---

## Reflection

### 1. What did you learn?
Developers value **clarity and stability** more than detailed documentation. Clear problem definition and consistent requirements significantly improve development efficiency.

---

### 2. How can PMs prevent miscommunication?
- Write clear and concise requirements
- Define acceptance criteria upfront
- Align early with developers before implementation
- Avoid changing scope mid-development

---

### 3. One change to improve documentation
Include **explicit acceptance criteria and edge cases** in every feature spec to reduce ambiguity and improve implementation accuracy.

---

## Key Takeaways
- Clear communication is more important than long documentation  
- Developers prefer well-defined problems over vague ideas  
- PMs should think from both **user and technical perspectives**  
- Good specs reduce rework and speed up delivery
